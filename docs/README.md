# docs

- `overview.md`：产品目标、范围和设计原则
- `frontend-design.md`：前端页面布局、路由、组件和状态设计
- `backend-design.md`：后端模块分层、服务边界与运行模型
- `integration.md`：前后端交互契约、实时通道与缓存策略
- `architecture.md`：系统架构、模块分层和实时通道设计
- `api.md`：REST、WebSocket、SSE 接口设计
- `security.md`：安全边界、审计和风险控制
- `testing.md`：测试策略、覆盖率门槛和功能测试要求
- `quality-gates.md`：CI、覆盖率和合并门禁要求
- `roadmap.md`：分阶段实施路线图
